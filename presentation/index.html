Introduction

Hello everyone. Today, we're diving into the world of MongoDB, a powerful NoSQL database. As front-end developers, you're accustomed to crafting engaging user experiences, but what happens behind the scenes is equally crucial. Let's explore how MongoDB fits into the backend ecosystem and why it's worth our attention.
Before delving into MongoDB, let's quickly recap what databases are. Think of them as organized collections of data. Traditionally, we've used relational databases like MySQL or PostgreSQL. They excel at structured data but can become rigid as applications evolve. Then non-relational databases can help us. What's the difference between them?
Relationships in a relational DB
A relational database uses tables to store data, where each table contains records with attributes. Relationships between tables are provided through keys. A non-relational database can use various data models such as document-oriented, key-value, columnar or graph. They do not require a strict schema and can store data in a more flexible form.

What is Mongo DB?

Documents are grouped into collections - analogous to tables in relational databases, but they do not have a strict schema, and documents in a collection can have different sets of fields. MongoDB is a leading NoSQL database, known for its document-oriented structure. Instead of tables, it stores data in flexible, JSON-like documents, making it a natural fit for JavaScript-heavy applications. MongoDB is free to use and published under the AGPL.

Let's discuss some key concepts:

  • Documents.
  • Collections.
  • Fields.

Documents

A basic unit of data in MongoDB, similar to rows in relational databases. Documents are a collection of key-value pairs, where the values can be simple data types (strings, numbers, booleans), arrays, or nested documents.

Collections

Groups of documents, similar to tables in relational databases, but without a fixed schema, and documents in a collection can have different sets of fields.

Fields

In MongoDB, fields are the names and values that make up documents in collections, comparable to columns in relational databases. Fields can have various data types such as strings, numbers, booleans, arrays, nested objects, dates, and others.

As frontend developers, why should we consider MongoDB?

  • JSON-Like Documents.
  • Scalability.
  • Performance.
  • Flexibility.

JSON-Like Documents:

Data stored in a format we're already familiar with, simplifying development. MongoDB was designed from its inception to be a database focused on delivering great development experience. JSON’s ubiquity made it the obvious choice for representing data structures in MongoDB’s document data model.

Scalability:

MongoDB's distributed architecture allows us to seamlessly scale as our application grows, allowing us to distribute data across multiple nodes and provide better performance as our workload grows.

Performance:

Its document-oriented model often results in faster query execution, improving the user experience, especially if the data is stored denormalized and does not require complex joins.

Flexibility:

There is no need to change the database schema when requirements change, which increases flexibility since adding new attributes or changing table structures can be a complex and expensive process.

Use Cases:

  • Content Management Systems (CMS).
  • Mobile Applications.
  • Payments.

CMS:

Easily manage diverse content types without schema migrations. With MongoDB, you can store and serve any type of content, build any feature, incorporate any kind of data in a single database. Faster. With less money.

Mobile Applications:

Sync data seamlessly across devices, even in occasionally connected environments. As consumers demand seamless mobile experiences, choosing the right underlying technology to handle your mobile data is essential.

Payments:

Enrich the payments experience by consolidating and analyzing data in any format in real time, driving value-added services and features to consumers. Accept and enrich any payment data type. Eliminate downtime and outages with distributed fault tolerance and automated data recovery capabilities. Stay competitive and lower TCO with composable payment systems that combine best-of-breed products from third parties.

Integration with Web Technologies:

  • Node.js
  • Express.js
  • React, Angular

Node.js:

Native drivers enable smooth interaction between MongoDB and Node.js applications.

Express.js:

Popular Node.js framework for building web applications, with robust MongoDB support. For all your data storage concerns, you can easily integrate MongoDB with the native driver available to you. Using this stack, you can leverage MongoDB's document model with the REST API standard payload, which uses the JSON format.

React, Angular:

Front-end frameworks that leverage MongoDB's JSON-like data structures effortlessly. React is part of the MERN stack, which includes MongoDB, ExpressJS, React, and NodeJS. The advantage of this stack is a single programming language – Javascript. Angular is part of the famous MEAN stack, which also consists of MongoDB, ExpressJS and NodeJS. Like the MERN stack, it relies entirely on JavaScript for both the front-end and back-end.
